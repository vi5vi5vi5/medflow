# services/

Бизнес-логика. Сервисы принимают «голые» модели из
[`../models`](../models) или JSON, валидируют, проверяют доменные
инварианты, дёргают репозитории из
[`../interface/Abstract`](../interface/Abstract) — и возвращают
объекты или бросают `std::invalid_argument` / `std::out_of_range`
для ошибок ввода.

API-слой ([`../api`](../api)) ловит эти исключения и превращает
в 400-ответы. Это упрощает контракт: сервис не знает про HTTP, не
знает про JSON-формат ответа, не знает про конкретный репозиторий —
только модели и интерфейсы.

## Сервисы

| Файл                        | За что отвечает |
|-----------------------------|-----------------|
| `ClinicService.h`           | CRUD клиник + каскадные удаления (комнаты, расписания, пациенты — отвязка/удаление). |
| `RoomService.h`             | CRUD кабинетов, проверка принадлежности клинике. |
| `SpecializationService.h`   | CRUD специальностей. |
| `DoctorService.h`           | CRUD врачей, проверка специальности, поиск, каскад при удалении. |
| `PatientService.h`          | CRUD пациентов, валидация полиса через [`InsuranceValidator.h`](InsuranceValidator.h), каскад при удалении (история приёмов). |
| `ScheduleService.h`         | Расписания: проверка занятости комнаты и врача, вычисление свободных дней/слотов для записи пациента. |
| `AppointmentService.h`      | Создание/изменение/удаление приёмов, проверка слотов через `ScheduleService`, переходы статусов. |
| `BackupService.h`           | Экспорт/импорт всей БД в один JSON (бэкап = полная точка восстановления; `importAll` сначала чистит БД, чтобы не было наложений). |
| `InsuranceValidator.h/.cpp` | HTTP-валидация номера полиса через внешний сервис (требование заказчика). `.cpp` отдельно, чтобы httplib не тянул windows-заголовки в `PatientService.h`. |

## Соглашения

- Сервисы — **header-only** (кроме `InsuranceValidator.cpp` —
  см. ниже).
- Принимают зависимости через `shared_ptr<I*>` в конструкторе,
  ничего не создают сами.
- На «неправильный ввод» бросают `std::invalid_argument` с
  человекочитаемым сообщением (часто на русском — оно уйдёт
  пользователю в `{"error": "..."}`).
- На «ничего не нашёл» возвращают `std::optional<>` или пустой
  `vector<>` — это не ошибка.

## Почему `InsuranceValidator` отдельный `.cpp`

`PatientService.h` хочется держать чистым заголовком, но он зовёт
HTTP-валидатор, а `httplib.h` на Windows тянет `<winsock2.h>`. Если
включить httplib прямо в `PatientService.h`, ниже по цепочке
включений приходит `using namespace std;` из моделей — и ловим
неоднозначность `std::byte` vs `::byte` в COM-заголовках. Поэтому
HTTP-код живёт в отдельной translation unit, где httplib включается
первым и в изоляции.
