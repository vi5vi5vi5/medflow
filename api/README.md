# api/

HTTP-слой приложения. Один файл — один сервер на [cpp-httplib](../requirements/httplib.h),
который раздаёт REST-эндпоинты и статику из [`../web`](../web).

## Файлы

- **`ApiServer.h`** — публичный интерфейс (конструктор, `run()`, `stop()`).
  Зависит от сервисов из [`../services`](../services), но не от конкретных
  репозиториев.
- **`ApiServer.cpp`** — реализация. Внутри `struct Impl` с группами
  `setupPatients()` / `setupDoctors()` / … / `setupBackup()` /
  `setupMisc()` — по эндпоинту на ресурс.

## Соглашения

- Все JSON-ответы и ошибки идут через хелперы `ok(res, json)` и
  `err(res, status, msg)` — единый формат `{"error": "..."}` для 4xx/5xx.
- Исключения сервисного слоя ловятся в `set_exception_handler`:
  `invalid_argument`/`out_of_range`/`json::exception` → 400,
  всё остальное → 500. Бросать `throw` из сервиса — нормальный путь
  вернуть 400 пользователю.
- CORS открыт настежь (`*`), потому что фронт ходит с того же origin —
  если будут отдельные домены, ужать.
- `httplib.h` обязан включаться **первым** (см. комментарий в начале
  `ApiServer.cpp`): он тянет `<windows.h>` и без этого ниже ловится
  неоднозначность `std::byte` vs `::byte`.
