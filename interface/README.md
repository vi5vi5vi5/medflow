# interface/ — слой персистентности

Репозитории: абстрактные «контракты» хранения и их конкретные реализации.
Это граница между бизнес-логикой и тем, *где* реально лежат данные.

```
interface/
├── Abstract/    чистые интерфейсы (virtual) — то, на что опирается сервисный слой
├── InMemory/    хранение в std::unordered_map — для тестов и быстрых проб
└── sqlite/      боевая реализация поверх SQLite (её собирает main)
```

## Зачем разделение (порты и адаптеры)

Сервисы из [`../services`](../services) принимают только абстракции —
`shared_ptr<IPatient>`, `shared_ptr<IClinic>` и т.д., — и **никогда** конкретный
`Sqlite*`/`InMemory*`. Конкретные реализации подставляются в одном месте, в
точке сборки [`../MedFlow_BY_LLl-23.cpp`](../MedFlow_BY_LLl-23.cpp). DI-фреймворка
нет — граф зависимостей собирается руками через `make_shared`.

Что это даёт:

- **Прод:** в `main` всё собрано на `Sqlite*` поверх одного общего `SqliteDb`.
- **Тесты/пробы:** можно подсунуть `InMemory*` — без диска, без схемы, без миграций.
  Сервис не заметит разницы.
- **Расширение:** Postgres / ORM / что угодно — это новая папка-сосед, реализующая
  те же интерфейсы. Сервисы и API не трогаются.

## Один интерфейс — одна сущность

По одному репозиторию на доменную структуру из [`../models`](../models):
`Clinic`, `Room`, `Specialization`, `Doctor`, `Patient`, `Schedule`,
`Appointment`. Имена — `IClinic`, `IRoom`, … (интерфейсы), `SqliteClinic` /
`InMemoryClinic` (реализации).

## Что где смотреть

| Нужно | Куда |
|---|---|
| Сигнатуры и контракты (`save`/`findBy`/`removeBy`) | [`Abstract/`](Abstract) |
| Реальные SQL, схема таблиц, общий коннект | [`sqlite/SqliteDb.h`](sqlite/SqliteDb.h) + [`sqlite/`](sqlite) |
| Как сущности живут «без БД» (id-генерация, поиск по предикату) | [`InMemory/`](InMemory) |

## Важное про целостность

Репозитории **тупые**: они не валидируют связи между сущностями и не делают
каскадов. Существование клиники при создании кабинета, каскадное удаление
расписаний врача, запрет пересечения приёмов — всё это живёт в сервисном слое
([`../services`](../services)). Даже в SQLite-схеме нет внешних ключей (подробнее —
в [`sqlite/README.md`](sqlite/README.md)). Репозиторий просто пишет и читает строки.
